Hexo搭建属于自己的博客

发表于 2018-11-02 更新于 2019-02-01 分类于 bolg搭建

封面是我的灵魂绘图

10月下旬一个中午，隔壁宿舍同学发来了自己建的博客网址，点开发现很有意思，想学习一下，发现网上有的资料很多，傻瓜教程也很多，在四天三夜之后基本建好了，作为一个懒人实在是不想写一篇傻瓜教程，因为网上这方面的资料已经很多了，但是又想留一个参考给以后的自己查阅，顺便整理一下这一路来的思路，中间因为学习方面的事情耽误了一个周，赶上这个月的四天小长假，决定抓住时机把这篇文章补上。

下面我主要分成四部分：

  • 你为什么要搭博客
  • 怎么搭一个Hexo框架的博客
  • 怎么添加博客的自定义效果

不想看废话的可以直接跳到想看的章节。

为什么要搭博客

回答这个问题之前，要先想想另一个问题：

为什么要写博客

博客，在如今信息碎片化快餐化的时代，我个人感觉它几乎被埋没了。而且如今自媒体早已从博客转战更加方便快捷且传播力更广的微信公众号上了，所以，现在坚持写博客的人，大多是在“孤芳自赏”吧。

但是搭建博客后有几个我个人认为非常实用的功能：

  1. 博客可以是一个私人网络记事本

  2. 从专业的角度来讲，博客是一个巨大的参考库，每次我在学习上遇到问题时，搜索结果中给出最靠谱答案的地方只有三处：

    • 官方文档
    • 相关论坛
    • 技术博客

    其中官方文档给出的必然是标准答案，但是往往写的生涩难懂，而且平时遇到的很多问题都是超出官方考虑范围的；而论坛上答主给出的解决方案也很正确，但是好点的论坛大都是英文的，英语太渣的话光是描述你的问题就已经很困难了，何况是理解他人给出的答案，再加上国内用户量规模巨大的优质论坛太少，面向专业也不够专一，所以很难搜到满意的答案；而在CS领域，程序员们有个爱好：写技术博客，曾经和程序猿朋友交流过，他把博客比作一本字典:

    在遇到问题解决问题以后，程序员习惯将整个过程通过博客记录下来，方便以后自己突然忘记了，就翻出来看看，如今的markdown语言排版十分便捷，而且对代码展现十分友好，技术博客一直还可以。

    在技术博客中，博主会清晰的表述自己在操作整个过程中的思路和遇到的问题，你会很容易吸收和理解。

    举个不恰当的例子：我在写一份试卷，写完之后，我先去找标准答案（官方文档），发现有些题我做错了，但是我不知道我哪里做错了，参考答案上没有给写出细致的解题步骤。这时我就要先去问同班同学（论坛），大家一起相互讨论，但是发现同学们要不就是错的和我不一样，要不就是懒的给我解释我错在哪。最后，我只能求助上届学长（技术博客），学长果然老司机，当年的考试题都写在了本子上，满满的解题步骤从头到尾滴水不漏，你一看便发现了自己错在哪里了，问题解决。

  3. 在工作方面，博客可以作为面试的加分项。

回到第一个问题，为什么要搭blog，现在的第三方blog平台很多，比如csdn、简书，LOFTER等等。虽然，第三方blog平台存在着广告，自定义程度极弱，数据可能不安全的问题，但是对于重视内容创作的博主来讲，第三方blog有着超级人性化的UI，提供了庞大的用户基础，可以让自己的文章让更多的人看到，更容易推广出去。如果不想费劲捯饬博客，使用第三方平台是个不错的选择，但是如果你吃饱了没事干，或者你是一个计算机专业的从业人员，尝试一下自己搭建博客是很有必要的。就拿我个人的搭建经历来说，在使用HEXO的过程中，我从一无所知，逐渐认识了html+css+js的前端语言，了解了json格式的配置文件，认识了markdon这种快速书写格式的标记语言、GitHub这种开源的代码分享存储网站。这些工具和经验，或多或少都会在以后的工作中碰到，提前认识一下不是件坏事。

怎么搭一个Hexo框架的博客

本着不写傻瓜教程的懒人原则，我只写下各个步骤，然后留下具体教程的网站。其实所有的东西，在官方的配置文件中都写的明明白白，只要英语不差，仔细阅读，并不需要什么高深的逻辑和数学知识，别想的太难。

学会上外网

想要快速搭blog可以跳过此步，不过还是强烈建议大学生们上学掌握的第一项技术就是“翻墙”。光是一个Google搜索引擎所带来的收益绝对够回你租vps的本钱。

核心思路：

  • 租借一个虚拟专用服务器（Virtual Private Server），下面简称VPS

    这个教程很多，步骤很简单，我就不附上网址了，推荐国外的搬瓦工，官方网址https://bandwagonhost.com/

  • 去GitHub下载Shadowsocks，下面简称ss

  • 配置VPS的各种参数，在VPS（服务器端）上安装好ss，具体教程参考：https://blog.huihut.com/2016/12/03/BandwagonShadowsocksServer/

    这一步是最不容易理解的，以前搬瓦工的服务器是自带安装ss服务的，现在没有了，需要在VPS上手动输入命令来安装，具体在教程里也有写。

  • 打开你电脑上下载好的ss，配置服务器，将ip、端口、密码等等输入后，即可使用了。

搭建HEXOblog

如果你完成了第一步，那么下面的工作你完全可以自己完成了，去Google把我的标题搜索一下就可以了，我只能说我是懒人中的懒人，在此我附上另一个懒人整理的别人搭blog的教程：http://jmyblog.top/Hexo-GithubPages-CodingPages%E6%90%AD%E5%BB%BA%E8%87%AA%E5%B7%B1%E7%9A%84%E4%B8%AA%E4%BA%BA%E5%8D%9A%E5%AE%A2/

全文写的十分详细，我只把提纲提炼出来，抓几个点说一说：

1.获得个人网站域名

就是申请一个自己的网址，其实在你搭建blog，部署blog的过程中，GitHub或者coding会给你分配一个域名，但是如果你想有一个自定义后缀的域名，需要你花钱去买，这就是第一步的作用，个人感觉是锦上添花的事情，如果你有闲钱，可以买个自己喜欢的域名。

2.GitHub创建个人仓库

这步很简单，其目的是为了让你把本地的hexo的整个项目部署到GitHub上，然后，通过GitHub的page功能，在你每次访问你的项目网址时，GitHub便会把你的hexo项目渲染，以网页的形式呈现。blog就搭建成功了

3.安装Git

4.安装Node.js

其实这两个步骤我当时是完全按照傻瓜教程做的，通俗的讲，Git的作用是将本地的hexo同步到GitHub上，Node.js是一种js开发运行环境，Hexo是基于此制作的。所以安装hexo之前要先把它安装好。

5.安装Hexo

Hexo网站现在已经有中文版本，在安装之后，你可以尝试学习一些基础的命令行。

6.推送网站

这里需要去hexo的安装目录下找到_config.yml 文件，这个文件在之后的自定义环节也会用到，在主题网站Next上也会详细的介绍站点配置文件与主题配置文件。打开之后会发现里面的参数很容易理解，后面自定义的很多工作，大部分都是在这里（比如添加2Dlive、更改主题文件、添加本地搜索，添加emoji表情等等）和主题配置文件里完成的。

7.绑定域名

8.在coding或GitHub上部署你的网站。

coding是被腾讯收购的一个类似GitHub的网站，如果不翻墙的话，访问的速度要比GitHub快很多。我是把GitHub和coding都部署了，这样每次发blog时，如果出错了，可以通过对比两个网站来排查问题。

写blog，发blog

其实到这里，你就可以开始写你的blog了，使用hexo的基本命令去创建md文件，然后使用md语言来写文章，使用hexo命令生成html文件，然后部署到GitHub上，就可以在浏览器里，输入你的博客网址，查看你的blog了。

但是之所以搭blog，为的是就是自定义功能，把自己的网址做的很个性。

如何自定义blog

自己搭建的blog个人认为最大的好处有两个：

  • 相对的安全：数据全在本地电脑上，每次修改更新，都需要从本地上传，别人能篡改你blog的可能性几乎为0
  • 超级强大的自定义功能：你的blog说白了，只是几个文件。然后通过渲染器的渲染，呈现出你所访问的网页的效果，对于一个网页，离不开html+css+js的格式，对于第三方blog平台，这些文件都是固定的一般禁止用户去访问修改，但是对于你自己搭建的blog，这些文件全部存储在本地，你完全可以自由发挥。

我打个不恰当的比方：第三方blog平台是别人盖好装修好的房子租给你，允许你在里面吃喝拉撒，但是不能动房子的一砖一瓦；但是hexo是别人给了你一个毛坯房，你可以在里面放飞自我的装修，加这个加那个，但前提是不能把房子拆了重新改，房子的整体结构不能变；而还有一种就是纯粹的从零开始盖房子，所有的东西都要自己造，这种行话叫“造轮子”，造轮子其实就是在重复一些十分低级且工作量巨大的工作，这在程序员之前是极不推崇的。

自定义过程的经验总结

在自定义中，有很多功能是配置文件中已经自带的，代码也都写好了，只需要把代码行的注释符号去掉，或者把参数改成on即可，其中评论功能比较麻烦。来必力不错，但是反应有点慢，而且风格和我的主题不搭，最后我选择了gitment，但是gitment需要在GitHub中配置一个项目来存评论，比较麻烦。然后再是mathjax选项，为了让blog支持数学公式，必须打开配置文件中的mathjax功能，然而，有些符号在markdown中会和html冲突，这里需要更换渲染器，我最终找到了格式切换的瑞士军刀–pandoc，pandoc几乎支持所有格式的相互转化，在GitHub中找到一个pandoc类型的渲染器，然后用命令行把原本的渲染器换掉。2d-live功能（就是我blog右下角的小猫咪）是GitHub上的一个开源项目，next并不自带，官方文档也有中文的，感兴趣的也可以自己去琢磨一下。
